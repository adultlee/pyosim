# 쌍둥이 득표 — 정의 및 카드 표시 스펙

> 확정일: 2026-06-17. 이 문서가 쌍둥이 득표의 단일 기준이다. 코드/UI는 여기에 맞춘다.

## 1. 배경

2026-06-03 9회 지선 인천 송도1·2동 관내사전투표에서 양당 후보 득표가 완전 일치한 논란("5.9억분의1 확률" 주장)에 대해, **"과거 선거에서 이런 일이 얼마나 자주 있었나"를 빈도로 보여주는** 중립적 맥락 도구다. 부정선거 탐지가 아니며, 확률 계산은 하지 않는다.

## 2. 정의 (★ 핵심)

같은 **비교 단위** 안에서, **서로 다른 두 투표소(읍면동)에서 같은 두 후보가 같은 득표수**를 받으면 쌍둥이다. = **투표소 "간" 반복.**

예시:
```
A동 관내사전투표:  박찬대 583  =  유정복 583
B동 관내사전투표:  박찬대 583  =  유정복 583   ← 서로 다른 두 동에서 같은 일이 반복 → 쌍둥이
```

### 판정 규칙
- **순위 무관**: 한 투표소 내에서 인접한 순위 쌍이면 됨 (1·2위, 2·3위, 5·6위 …).
- **최소 득표 임계값: 각 후보 10표 이상**인 동률만 인정. 1표=1표 같은 군소후보 저득표 동률은 노이즈이므로 제외. ← **핵심 품질 기준.**
- **이름 + 득표수 둘 다 일치**해야 함. (표수만 같고 후보가 다르면 쌍둥이 아님.)
- **같은 투표 구분(level)끼리만** 비교. 관내사전투표는 관내사전투표끼리, 선거일투표는 선거일투표끼리.
- **합계(`계`) 행은 입력에서 제외.** 사전+선거일이 우연히 합산 동률이 되는 가짜 사례를 막기 위해.

### 최소 단위 = 읍면동 × 투표구분 (★ 통일 필수)

비교의 최소 단위는 **읍면동 × 투표구분(level)**이다. 개별 투표소/투표구로는 내려가지 않는다.

- **회차마다 원본 단위가 다르다** — 3회 지선은 투표구(`가리봉제1동제3투`)까지 쪼개져 있고, 9회·총선·대선은 읍면동(`괴정제3동`)까지만 있다.
- 단위가 다르면 비교가 불공정하다(작은 단위일수록 득표가 작아 동률이 잘 남 → 3회 과대계상). 그래서 **모든 회차를 읍면동으로 통일**한다.
- 투표구는 읍면동으로 **합산**한다. 추출 규칙: 읍면동명 끝의 `제N투`, `투표소` 접미사 제거 (`삼례읍제8투`→`삼례읍`, `강일동투표소`→`강일동`). 예외 케이스(~30개)는 점검.
- analyze_twin_votes.py에 `_clean_eupmd_local`/`_restore_middot` 정제 함수가 이미 있으니 토대로 활용.
- 시·구 단위로는 절대 올리지 않는다(득표수가 커져 동률이 사실상 안 나옴).

### 비교 단위 (group key)
| 선거 | 단위 |
|---|---|
| 대선 | (회차, level) — 전국 |
| 총선 지역구 | (회차, 선거구분, 선거구명, level) |
| 총선 비례 | (회차, 선거구분, level) — 전국 |
| 지선 시도지사·교육감·광역비례 | (회차, 선거종류, 시도그룹, level) |
| 지선 나머지 | (회차, 선거종류, 시도, 구시군, level) |

### 혼동 주의 (실제로 했던 실수)
- ✗ "투표소 **내** 1·2위 동률"(한 투표소에서 1위=2위)과 혼동 금지. 이 도구는 투표소 **간** 반복이다.
- ✗ `계` 합계 행을 투표소로 착각 금지. 가짜 동률이 대량 잡힌다.

## 3. 카드에 보여줄 값

카드 1장 = 하나의 쌍둥이 패턴(같은 두 후보·같은 득표수가 여러 투표소에서 반복된 것).

### 표시 항목
1. **동률 후보 (주인공)** — 각 후보를 `정당 + 이름`으로, **정당색 배지**와 함께.
   - 예: 🔵 더불어민주당 박찬대 **583표**  =  🔴 국민의힘 유정복 **583표**
2. **투표 규모** — 그 동률이 전체 대비 얼마나 큰지 판단할 수 있게 **투표수 / 선거인수**를 함께 표시.
   - 예: `583표 (투표수 1,247 중)`
3. **반복 횟수** — 이 패턴이 나타난 투표소 수 (count). 예: `22곳에서 반복`
4. **순위 쌍** — 몇 위·몇 위의 동률인지 (rank_pair). 예: `1·2위`
5. **맥락** — 회차 + 투표구분 + 지역. 예: `지방선거 9회 · 관내사전투표 · 경기도`
6. **펼치기** — 반복된 투표소 목록(locations) 토글.

### 정당색 매핑 (계열색 통일)

역사적 정당은 당시 공식색이 아니라 **계열색으로 통일**한다. 데이터에 정당이 278종(대부분 군소)이므로, 주요 계열만 색을 주고 나머지는 회색 폴백.

| 계열 | 색 | 포함 정당(예) |
|---|---|---|
| 민주당 계열 | `#152484` (파랑) | 더불어민주당, 열린우리당, 민주통합당, 새정치민주연합, 통합민주당, 새천년민주당 |
| 보수 계열 | `#E61E2B` (빨강) | 국민의힘, 새누리당, 한나라당, 자유한국당, 미래통합당 |
| 개혁신당 | `#FF7210` (주황) | 개혁신당 |
| 정의·녹색 계열 | `#FFB500` / `#0073CF` | 정의당, 녹색정의당, 노동당, 진보신당 |
| 진보당 | `#D6001C` (자주) | 진보당, 통합진보당, 민주노동당 |
| 국민의당/안철수 계열 | `#06B4AA` (민트) | 국민의당, 국민의미래 |

- 매핑은 정당명 부분일치/별칭 테이블로 구현. 새 정당이 나와도 **폴백(회색)** 으로 안전하게 처리.
- 후보 데이터에 정당이 없으면(현 JSON 상태) 배지 없이 이름만 표시 — 하위호환.

## 4. 데이터 요구사항 (현재 JSON에서 추가 필요)

현재 `web/public/twin_votes_*.json`은 카드 스펙을 충족하지 못한다. 분석/파싱 단계에서 아래를 채워야 한다.

| 필요 값 | 현재 상태 | 비고 |
|---|---|---|
| 후보 **정당** | ❌ 이름만 있음 | 원본 헤더 `더불어민주당정원오`에서 정당 분리 필요 |
| **투표수 / 선거인수** | ❌ 버려짐 | 원본에 컬럼 있음 |
| 각 후보 **10표 이상** 필터 | ❌ (0표만 제거됨) | analyze 단계에서 적용 |
| 동률 후보 2명 모두 votes에 | ⚠ 일부 1명만 보임 | 위 임계값/구조 점검 시 함께 |

→ 이 값들은 **입력 CSV(`data_processed/`, 다른 데스크탑 보유)** 로 재생성해야 반영된다. CSV가 정당·투표수 컬럼을 보존하는지도 확인 필요.
