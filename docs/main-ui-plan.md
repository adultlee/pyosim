# 메인 UI 기획안

> 작성: 2026-06-17. 새 정의([twin-vote-spec.md](twin-vote-spec.md))를 반영한 메인 화면 재설계.

## 현재 화면 구조 (App.tsx)

```
[헤더]  표심 · 쌍둥이 득표 분석
[제목+설명]  쌍둥이 득표란? + 송도 논란 배경 박스
[선거 종류 탭]  지방선거 | 총선 | 대선
[회차 탭]  3회(2,747) | 4회(133) | ... | 9회(136)
[TwinVoteViewer]
  ├ 필터 바: 투표 구분 / 선거 종류 / 순위 쌍
  ├ "N개 그룹 · 두 후보 합 득표 기준 정렬"
  └ 테이블: 그룹헤더 + 투표소 행들 (100개씩 페이지)
```

## 현재 화면의 문제점

1. **설명문이 옛 정의** — "득표 임계값은 두지 않으며"는 폐기됨(이제 각 후보 10표+). 정의 자체도 "투표구에서 이름·득표 동일"이라 모호.
2. **정렬 기준이 약함** — "두 후보 합 득표"순. 사용자가 주목하는 건 ① 득표 규모 ② 반복 횟수(count)인데 합 기준은 직관과 어긋남.
3. **카드가 아니라 빽빽한 테이블** — 정당색·투표규모 같은 새 카드 스펙을 담기 어려운 구조. 한 그룹이 수백 행이면 스크롤 지옥.
4. **회차 건수가 단위 불일치로 왜곡** — 3회(투표구 단위)가 과대. 읍면동 통일 후 재계산 필요.
5. **노이즈가 상단 점유** — 1표=1표가 합 기준 정렬에서 위로 올 수 있음(10표 필터로 해소 예정).

## 재설계 방향

> **디자인 확정 (2026-06-17 브레인스토밍 결과).** 비주얼 컴패니언으로 합의됨. 목업은 `.superpowers/brainstorm/`.

### A. 헤더/인트로
- 제목 + **한 줄 정의**(새 기준): "서로 다른 두 읍면동에서 같은 두 후보가 같은 표를 받은 사례".
- 배경 박스(왜 보는가)는 **유지**. 송도 수치(박찬대 3,030 등)도 유지 — 맥락 설명에 필요.

### B. 네비게이션 (선거종류 → 회차)
- 현행 탭 유지. 단 회차 건수는 **읍면동 통일 후 재계산된 값**.

### C. 결과 영역 — 테이블 구조 **유지**, 그룹 카드 + 3컬럼 그리드로 다듬기

카드로 갈아엎지 않는다. 기존 "그룹 헤더 + 투표소 목록" 구조를 유지하되, 그룹을 둥근 카드로 감싸고 반복 투표소를 **3컬럼 그리드**로 배치한다.

```
┌─ 그룹 카드 ────────────────────────────────────────────┐
│ ● 더불어민주당 박찬대  3,030   =   1,440  유정복 국민의힘 ●  │ ← 동률 헤더(득표 큰글씨)
│ 관내사전투표 · 1·2위 · 인천 연수구 · 9회 지선 · 2곳에서 동일 │ ← 메타 한 줄
│ ┌──────────┬──────────┬──────────┐                      │
│ │📍 송도1동 │📍 송도2동 │  (빈칸)   │ ← 3열 그리드            │
│ │3,030=1,440│3,030=1,440│          │                      │
│ │투표 4,470중│투표 4,470중│          │                      │
│ └──────────┴──────────┴──────────┘                      │
│ (반복 많으면) 6칸 노출 + "나머지 N곳 더보기 ▾"            │
└────────────────────────────────────────────────────────┘
```

- 투표소가 적으면 채워진 칸만, 많으면(예 22곳) 3열로 가지런히 + 앞 6칸 + 더보기.
- 각 칸: 읍면동 · 지역 · 해당 득표(`3,030 = 1,440`) · (있으면) 투표 규모.

### D. 배지 — "모두 배지 ✗", 위계 차등
- **반복 횟수** = 강조(민트). 반복 많을수록 진한 색, 적으면 옅게. (가장 눈에 띄게)
- **투표구분** = 보조 표시. 관내사전투표는 강조(논란 핵심), 선거일은 차분.
- **정당** = 계열색 점 + 정당명. 과하지 않게.
- → 셋을 같은 무게 배지로 나열하지 않는다. 반복 횟수만 진짜 배지, 나머지는 색/텍스트로 낮춤.

### E. 정렬·필터
- **기본 정렬**: 득표수 큰 순(확정). 보조로 "반복 횟수 순" 토글.
- **필터**: 투표 구분 · 순위 쌍은 **드롭다운으로 접기**(현재 91·92위까지 7줄 칩 → "순위 ▾" 하나) · 최소 득표(10표 하한).
- 페이지네이션 유지.

### F. 정당색 (twin-vote-spec.md §3 색표)
- 계열색 통일. 정당 데이터 없으면 색 점/배지 없이 이름만(하위호환).

## 구현 순서 (데이터 독립 = 지금 가능)

UI는 데이터 재생성과 **독립적으로 미리 구현·커밋 가능**. 새 필드(정당·투표수)는 "있으면 표시, 없으면 생략"으로 하위호환되게 짠다.

1. **정당색 유틸** (`partyColor.ts`) — 정당명→계열색 매핑 + 회색 폴백.
2. **순위 쌍 필터 드롭다운화** — 칩 7줄 → "순위 ▾" 하나. (가장 큰 시각 개선)
3. **그룹 카드 + 3컬럼 그리드** — TwinVoteViewer의 그룹 렌더를 둥근 카드 + 투표소 3열 그리드로. 앞 6칸 + 더보기.
4. **배지 위계** — 반복 횟수만 강조 배지(민트), 투표구분·정당은 낮춤.
5. **App.tsx 인트로** — 정의 문구 새 기준으로 교체. 배경 박스·송도 수치 유지.
6. (데이터 준비되면) 정당·투표수 필드 채워 재생성 → 정당 배지·규모가 자동으로 채워짐.

## 확정 사항 (브레인스토밍 결과)
- 구조: **테이블 유지** (카드 전면 전환 X). 그룹을 카드로 감싸고 투표소는 3컬럼 그리드.
- 정렬 기본값: **득표 큰 순**.
- 인트로 배경 박스 + 송도 수치: **유지**.
- 배지: **반복 횟수만 강조**, 정당·투표구분은 낮춤(모두 배지 금지).
- 세부 디테일(칸 패딩·빈칸 처리·더보기 임계 등)은 구현 재량.

## 데이터 의존성
- 정당 배지·투표수 표시는 JSON에 해당 필드가 있어야 동작 → 현재 없음(이름만).
- UI는 **하위호환**으로 먼저 구현(필드 있으면 표시, 없으면 생략). 데이터 재생성(다른 데스크탑 CSV)은 별도 작업.
- `types.ts`에 정당·투표수 옵셔널 필드 추가는 UI 구현 시 함께.
