# 후보쌍 모아보기 (Candidate-Pair Grouping) 설계

작성: 2026-06-17

## 배경

현재 쌍둥이 카드는 **득표값까지 고정된** 단위로 나열된다. 같은 두 후보(예: 민형배·이정현)가
여러 득표값으로 반복되면 서로 다른 카드로 흩어진다. 9회 지선 데이터 기준 837개 쌍둥이 그룹이
실은 76개 후보쌍에 흩어져 있다 (예: 김현욱·홍성규 = 106개 그룹, 일치 동 총 236곳).

사용자 요구: **같은 두 후보가 등장하는 사례를 한데 몰아서** 보여준다.

## 결정 사항 (확정)

- **묶음 기준:** 후보쌍 이름만 같으면 묶는다(득표값 무관). 단 기존 비교단위는 유지.
- **묶음 범위:** 기존 비교단위 안에서만 — `(회차, 선거종류, 지역, level, 순위쌍)`이 같아야 한 묶음.
  level을 가로지르거나 전국으로 합치지 않는다 (정의와 일관).
- **구현 위치:** 웹앱 런타임 집계(접근법 A). 분석 스크립트·JSON·types 무변경.
- **사례 줄:** 간결하게 — 득표값 + 일치 동 목록만. 정당색·순위는 후보쌍 헤더에만 표시.

## 묶음 키

```
묶음키 = category + group(시도/구시군/선거구명) + rank_pair + 후보쌍(이름 정렬)
```

기존 `EnrichedGroup`에서 `votes`(득표값)만 키에서 제외한다. 같은 후보쌍이라도 지역/level/순위가
다르면 별개 묶음이 된다.

## 데이터 변환 (런타임 useMemo)

```
EnrichedGroup[]  →  PairGroup[]

PairGroup = {
  key: string
  candidates: [string, string]        // 정렬된 후보쌍
  parties: Record<string, string>
  _electionType, _raceType, _level: string
  rank_pair: [number, number]
  group: Record<...>                  // 지역 메타 (사례 간 동일)
  cases: EnrichedGroup[]              // 득표값별 사례 (count 내림차순)
  totalLocations: number             // 모든 사례 count 합
  maxTotalVotes: number              // 사례 중 최대 두 후보 합 (정렬용)
}
```

## 카드 구조 (2단계)

```
┌─ PairCard ────────────────────────────────┐
│ ● 민형배   ·   ● 이정현      🔁 총 12곳·3사례 │  ← 후보쌍 헤더 (정당색 점)
│ 광주 시도지사 · 관내사전투표 · 1·2위          │  ← 메타 한 줄
├────────────────────────────────────────────┤
│  583 = 583   5곳    A동  B동  …              │  ← CaseRow (간결)
│  421 = 421   4곳    C동  D동  …              │
│  …나머지 사례 더보기                          │
└────────────────────────────────────────────┘
```

- **PairCard 헤더:** 후보쌍 이름(정당색 점) + "🔁 총 N곳·M사례" 배지 + 메타(지역/선거종류/level/순위).
- **CaseRow:** 한 사례(고정 득표값) = `득표값 = 득표값` + count + 일치 동 목록(그리드). 동이 많으면
  미리보기 후 더보기(기존 GRID_PREVIEW 로직 재사용).
- 사례가 많으면(예: 106개) 후보쌍 카드 자체에 "사례 더보기"로 일부만 우선 노출.

## 정렬·필터·페이징

- 필터(투표구분/선거/순위): 후보쌍 단위 적용. 한 후보쌍은 한 (category, rank) 안에 있어 자연 동작.
- 정렬: "반복 많은 순"(totalLocations) / "득표 큰 순"(maxTotalVotes). 기존 두 정렬 칩 의미 유지.
- 페이징: 후보쌍 단위. 76개 → 1~2페이지.

## 검증

1. `cd web && npm run build` (tsc + vite) 통과.
2. 9회 시도지사 사전투표에서 송도1·2동(박찬대 3030 = 유정복 1440)이 한 후보쌍 카드의 한 사례로 보임.
3. 김현욱·홍성규(106개 사례)가 한 후보쌍 카드로 접힘.
4. 기존 필터/정렬/페이징 동작 유지.
